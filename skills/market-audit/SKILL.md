# 全面营销审计 / Marketing Audit

## 触发方式
- "帮我做一个营销审计，网站是 https://xxx.com"
- "分析这个网站的营销做得怎么样"
- "全面诊断 https://xxx.com"

## Project Facts 门禁

先读 `skills/PROJECT-FACTS.md`，把官网抓取记录写入项目 `evidence`，再执行：

```bash
node scripts/project-facts.js gate --file output/{project}-project-facts.json --skill market-audit
```

无 GA4/GSC/CRM 证据时，本技能只做公开页面启发式审计，不声称掌握真实流量、排名、转化率或收入影响。

## 执行流程

### 第一步：发现
用 WebFetch 获取目标网站首页和关键页面（About、Pricing、Product、Blog），检测业务类型。

| 业务类型 | 判别信号 |
|----------|----------|
| SaaS | 免费试用CTA、定价页、功能页 |
| 电商 | 商品列表、购物车、类目 |
| 服务/代理 | 案例、作品集、联系表单 |
| 教育培训 | 课程列表、导师介绍、学员评价 |

### 第二步：6 维度分析
对以下维度分别评分（0-100）：

1. **内容与信息传达 (25%)** — 标题清晰度、价值主张、文案说服力
2. **转化优化 (20%)** — CTA 有效性、表单摩擦、信任信号
3. **SEO 可发现性 (20%)** — 标题标签、元描述、内容质量、技术 SEO
4. **竞争定位 (15%)** — 差异化、竞品对标、市场品类
5. **品牌与信任 (10%)** — 社交证明、About 页面、安全保障
6. **增长策略 (10%)** — 商业模式、定价策略、增长闭环

### 第三步：综合评分与建议

```
综合得分 = 内容×0.25 + 转化×0.20 + SEO×0.20 + 竞争×0.15 + 品牌×0.10 + 增长×0.10
```

评分等级：
- 85-100: A — 营销是竞争优势
- 70-84: B — 有明确的改进机会
- 55-69: C — 存在明显差距
- 40-54: D — 需要大幅改进
- 0-39: F — 营销严重不足

### 第四步：输出报告

生成 `output/MARKETING-AUDIT.md`，包含：
- 执行摘要（2-3段）
- 6维度详细分析（含评分、发现、建议）
- 快速行动清单（本周可完成）
- 战略建议（本月完成）
- 长期规划（本季度完成）
- 预计收入影响

## 数据来源标注（必填）

须遵守 `skills/DATA-SOURCE.md`：每个含**具体数字/评分/排名/KPI**的数据块下方追加一行 `*数据来源：...*` 或 `*基于 ... 估算/预估*`；报告末尾附 `## 数据来源汇总`。
