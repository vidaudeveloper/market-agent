# 邮件序列生成 / Email Sequence Generation

## 触发方式
- "帮我生成一套欢迎邮件序列"
- "为 XX 产品写邮件营销序列"
- "设计一套培育邮件的 drip campaign"

## 序列类型选择

| 类型 | 适用场景 | 邮件数 | 目标 |
|------|----------|--------|------|
| 欢迎序列 | 新订阅/下载资源 | 5-7 | 建立信任、传递价值、介绍产品 |
| 培育序列 | 温线索未准备购买 | 6-8 | 教育、建立权威、克服异议 |
| 上市序列 | 新品发布 | 8-12 | 建立期待、驱动购买 |
| 再激活 | 30-90天不活跃 | 3-4 | 赢回关注或清理列表 |
| 上手序列 | 新试用/新客户 | 5-7 | 驱动激活、减少流失 |
| 购物车放弃 | 电商放弃结账 | 3-4 | 挽回销售 |

## 核心原则：一封邮件一件事

每封邮件只有一个目的、一个CTA、一个期望动作。

## 邮件结构框架

### 价值先行框架（培育序列用）
```
第1封：纯价值（无推销）
第2封：纯价值（无推销）
第3封：价值 + 产品软提及
第4封：价值 + 案例展示产品效果
第5封：直接推销 + 紧迫感
```

### 故事驱动框架（单封邮件用）
```
Hook（2-3句故事/现象）
↓
桥梁（连接读者处境）
↓
洞察（可行动的教训）
↓
CTA（链接到下一步）
```

## 每封邮件包含
- 主题行（3-5个变体）
- 预览文字
- 正文（保持简洁）
- CTA 按钮文案
- 发送时机建议

## 数据来源标注（必填）

须遵守 `skills/DATA-SOURCE.md`：每个含**具体数字/评分/排名/KPI**的数据块下方追加一行 `*数据来源：...*` 或 `*基于 ... 估算/预估*`；报告末尾附 `## 数据来源汇总`。

## 输出
生成 `output/EMAIL-SEQUENCES.md`，含完整序列 + 每封邮件的主题行变体
