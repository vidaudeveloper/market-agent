# 客户提案生成 / Client Proposal Generator

## 触发方式
- "帮我生成一份客户提案"
- "为 XX 客户准备一份营销服务方案"
- "制作一份服务报价提案"

## Project Facts 门禁

先读 `skills/PROJECT-FACTS.md` 并执行 `gate --skill market-proposal`。提案只汇总同一 Project Facts 中的客户目标、诊断、竞品与证据。报价、KPI、ROI 必须区分用户确认值、平台实测值与公式估算值。

## 搜集输入
在生成前提问：客户名称与公司、行业模式、当前营销状况、痛点、目标、预算范围、决策时间线

### 如已有审计数据
自动从 `output/MARKETING-AUDIT.md` 中提取分析结果，融入提案，形成数据支撑的方案。

## 提案结构

### 1. 封面
```
营销策略提案
为：[客户名称]
准备者：[你的名字/公司]
日期：[日期]
```

### 2. 执行摘要（1页以内）
- 确认客户处境和目标
- 陈述核心问题你将解决
- 预览推荐方法

### 3. 现状分析
- 数据分析（如有审计数据）
- 与竞品的差距
- 增长机会量化

### 4. 推荐方案
- 服务范围（SEO/广告/内容/社媒等）
- 方法论和框架
- 时间线和里程碑
- 团队配置

### 5. 定价（三级选项）

| 方案 | 价格 | 内容 |
|------|------|------|
| 基础版 | ¥X/月 | [核心服务] |
| 标准版（推荐）| ¥X/月 | [+ 额外服务] |
| 旗舰版 | ¥X/月 | [+ 全部服务] |

定价心理学：用"标准版"做锚定，标注"推荐"

### 6. ROI 预估
用具体数字展示预期回报：
```
当前：月流量 X → 转化率 Y% → 收入 ¥Z
改进后：月流量 X+30% → 转化率 (Y+1)% → 收入 ¥Z'
月增量收入：¥XX,XXX
年化 ROI：XXX%
```

### 7. 案例/证明
- 过往项目成果
- 客户评价

### 8. 下一步
明确的行动号召和时间节点

## 数据来源标注（必填）

须遵守 `skills/DATA-SOURCE.md`：每个含**具体数字/评分/排名/KPI**的数据块下方追加一行 `*数据来源：...*` 或 `*基于 ... 估算/预估*`；报告末尾附 `## 数据来源汇总`。

## 输出
生成 `output/PROPOSAL.md`
