# 数据来源标注规范（全局强制）

> 所有技能产出报告时必须遵守。Agent 在写入 `output/` 前逐条自检。
> 项目级证据先写入 `output/{project}-project-facts.json`，契约见 `skills/PROJECT-FACTS.md`；本文规范负责将其转成报告中的人类可读来源标注。

---

## 核心规则

1. **凡出现具体数据**（数字、百分比、评分、排名、金额、日期断言、KPI 目标、销量、流量等），在该数据块**紧下方**追加 **一行斜体小字** 来源说明。
2. **数据块**包括：表格、带数字的列表项、评分卡、估算段落、图表说明文字。
3. **纯定性描述**（如「文案语气偏活泼」）可不加；但一旦含「提升 15%」等量化表述，必须加。
4. 报告**末尾**必须附 `## 数据来源汇总` 章节，列出本报告所有来源清单。
5. **禁止**无来源的精确数字；无法核实时必须写「估算/预估」并说明依据。
6. 报告中的精确数字须能回溯到 Project Facts 的 `facts[].evidence_ids`；`verified` fact 不得无 evidence。
7. 页面截图是视觉证据，不自动证明截图外的数值、归因或趋势。

---

## 标注格式（Markdown）

统一用斜体一行，置于数据块下方、空一行：

```markdown
| 指标 | Q3 目标 |
|------|---------|
| 全渠道曝光 | 3 亿+ |

*数据来源：基于 OPPO 官方产品页与 Q3 营销节点规划推断，非平台实测数据。*
```

```markdown
**综合得分：78/100（B）**

*数据来源：依据 https://example.com 首页/关于页 WebFetch 内容，按本技能评分 rubric 人工（AI）评估，2026-06-29。*
```

```markdown
| 排名 | 店铺 | 日销量 |
|------|------|--------|
| 1 | XXX | 12,580 |

*数据来源：出海匠 chuhaijiang.com 店铺排行，scripts/chuhaijiang-fetch.js 抓取于 2026-06-29。*
```

## 来源类型与写法

| 类型 | 写法模板 |
|------|----------|
| **网页抓取** | `*数据来源：[页面名称] [URL]，WebFetch/页面抓取于 [YYYY-MM-DD]*` |
| **脚本/API** | `*数据来源：[工具/脚本名]，查询于 [YYYY-MM-DD]*` |
| **MCP 工具** | `*数据来源：[MCP 服务名] [指标名]，查询于 [YYYY-MM-DD]*` |
| **用户提供** | `*数据来源：用户提供的 [材料/说明]，[YYYY-MM-DD]*` |
| **行业估算** | `*基于 [依据A、依据B] 估算/预估，非一手实测数据*` |
| **模型推断** | `*基于公开信息与营销框架推断，非官方披露数据*` |
| **混合** | `*数据来源：官网抓取 + 行业基准估算（分项见各节脚注）*` |

## 文档末尾汇总模板

```markdown
## 数据来源汇总

| # | 数据项 | 来源类型 | 来源说明 | 日期 |
|---|--------|----------|----------|------|
| 1 | 产品卖点 | 网页抓取 | OPPO Reno16 官网 | 2026-06-29 |
| 2 | Q3 曝光目标 | 行业估算 | 基于同类机型 Campaign 基准 | 2026-06-29 |
| 3 | 竞品价格带 | 网页抓取 | vivo/荣耀官网 | 2026-06-29 |

> 本报告中标注为「估算/预估」的数据仅供策略参考，不构成销售或财务承诺。
```

## 飞书 / Word 导出

导出到飞书或 Word 时**保留**所有斜体来源行与「数据来源汇总」章节，不得删除。
