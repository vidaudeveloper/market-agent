# 文案优化 / Copywriting Analysis

## 触发方式
- "分析 https://xxx.com 的文案"
- "帮我优化这段文案"
- "这个页面的文案写得怎么样"

## 第一步：文案提取
用 WebFetch 获取页面，提取：
- 主标题 (H1)
- 副标题
- 首屏文案
- 所有 H2/H3 标题
- CTA 按钮文字
- 导航标签
- 社交媒体证据

## 第二步：标题分析（5秒测试）
新访客能否在5秒内理解这家公司做什么、服务谁？

标题评分五维度（0-10）：
- 清晰度 / 具体性 / 相关性 / 差异化 / 情感触发

## 第三步：标题公式库
用以下框架生成替代标题：
- **PAS**（问题-激化-解决）
- **4U**（有用-紧迫-独特-具体）
- **Before-After-Bridge**

## 第四步：CTA 文案评分

| 等级 | 示例 |
|------|------|
| 弱 | "提交"、"点击这里" |
| 中 | "注册"、"开始" |
| 强 | "开始我的免费试用"、"获取我的定制报告" |

## 第五步：声调一致性检查
- 正式度、情感度、复杂度、幽默度、权威度

## 第六步：前后对比
为每个关键位置提供当前文案 + 优化版文案，附修改理由

## 数据来源标注（必填）

须遵守 `skills/DATA-SOURCE.md`：每个含**具体数字/评分/排名/KPI**的数据块下方追加一行 `*数据来源：...*` 或 `*基于 ... 估算/预估*`；报告末尾附 `## 数据来源汇总`。

## 输出
生成 `output/COPY-SUGGESTIONS.md`
